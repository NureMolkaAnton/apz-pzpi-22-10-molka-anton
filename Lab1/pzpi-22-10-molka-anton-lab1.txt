Vision of the Solution
Vision Statement
Ця система надасть власникам сонячних панелей інноваційне рішення для моніторингу та оптимізації їхньої ефективності. Завдяки автоматизованому збору даних про температуру, продуктивність і споживання електроенергії, продукт дозволить уникати перегріву панелей, що негативно впливає на генерацію енергії, а також раціоналізувати споживання енергії. Уявлення про світ з цим продуктом полягає у створенні простого, доступного й ефективного способу управління відновлюваною енергетикою для кожного власника сонячних панелей. Ця система сприятиме зростанню ефективності сонячної генерації та зменшенню витрат завдяки автоматизації та аналітиці.
Major Features
Моніторинг температури панелей: Постійний збір даних для запобігання перегріву.
Відстеження виробництва електроенергії: Надання точних показників щодо продуктивності панелей.
Система сповіщень: Інформування користувачів про аномалії або проблеми.
Аналіз статистики: Збір та аналіз даних для підвищення ефективності генерації.
Раціоналізація споживання енергії: Рекомендації щодо оптимального часу для енергоємних задач.
Інтеграція з мобільними та веб-застосунками: Зручний доступ до даних у реальному часі через різні платформи.
Безпека даних: Захист інформації під час передачі та зберігання.
Assumptions and Dependencies
Залежність від апаратного забезпечення IoT: Коректна робота системи залежить від якості сенсорів для збору даних.
Інтернет-з'єднання: Надійне підключення до мережі необхідне для передачі даних на сервер.
Технологічна база: Використання сучасних технологій для забезпечення стабільності та масштабованості системи.




Scope and Limitations
<The project scope defines the concept and range of the proposed solution. It’s also important to define what will not be included in the product. Clarifying the scope and limitations helps to establish realistic expectations of the many stakeholders. It also provides a reference frame against which proposed features and requirements changes can be evaluated. Proposed requirements that are out of scope for the envisioned product must be rejected, unless they are so beneficial that the scope should be enlarged to accommodate them (with accompanying changes in budget, schedule, and/or resources).>
Scope of Initial Release
MF-1: Моніторинг та запис температури сонячних панелей у режимі, близькому до реального часу через IoT-пристрої, з частотою оновлення даних кожні 5 хвилин (IoT, Backend).
MF-2: Збір та передача даних про температуру та продуктивність на сервер через MQTT протоколи (IoT). 
MF-3: Серверна частина для обробки, зберігання даних і надання RESTful API для доступу до інформації про температуру та продуктивність (Backend).
MF-4: Мобільний застосунок (Android) для відображення даних з сенсорів, сповіщень про аномалії (наприклад, температура вище 60°C) та базової статистики продуктивності (Android).
MF-5: Веб-застосунок для адміністрування та зручного доступу до даних і статистики через браузер із підтримкою кросплатформного доступу (Web)

Scope of Subsequent Releases
MF-6: Базовий аналіз даних для надання рекомендацій щодо оптимального часу виконання енергоємних задач (наприклад, зарядка електромобілів у пікові години генерації) (Backend, Android, Web).
MF-7: Інтеграція ШІ для прогнозування продуктивності сонячних панелей на основі історичних даних про температуру, продуктивність і погодні умови (отримані через API OpenWeatherMap) з використанням рекурентних нейронних мереж (RNN) (Backend).
MF-8: ШІ для оптимізації споживання енергії шляхом аналізу патернів генерації та споживання за допомогою алгоритмів кластеризації (K-Means), з рекомендаціями щодо перенесення енергоємних задач на періоди максимальної генерації (Backend, Android, Web).
MF-9: ШІ для автоматичного виявлення аномалій (наприклад, різке падіння продуктивності через пошкодження панелі) з використанням алгоритмів класифікації (Random Forest) і генерацією детальних звітів (Backend, Android, Web).
MF-10: Збір даних для ШІ, включаючи температуру панелей, обсяг виробленої електроенергії, погодні умови (температура повітря, сонячна радіація, вологість) і зворотний зв’язок користувачів через мобільний/веб-застосунок, із частотою оновлення кожні 5 хвилин (IoT, Backend).
Limitations and Exclusions
LE-1: Залежність від стабільного інтернет-з’єднання для передачі даних із IoT-пристроїв.
LE-2: Обмежена точність ШІ-прогнозів у перші 3–6 місяців через недостатній обсяг даних для навчання моделей.
LE-3: Виключена інтеграція з іншими системами управління енергією (наприклад, Smart Home платформи) через складність забезпечення сумісності та необхідність зосередитися на основній функціональності.
LE-4: Виключений багатокористувацький доступ для командної співпраці, оскільки цільова аудиторія – індивідуальні власники сонячних панелей, що зменшує складність розробки. 
LE-5: Виключений просунутий статистичний аналіз (наприклад, кореляція продуктивності з мікрокліматичними умовами) для збереження простоти інтерфейсу для кінцевих користувачів.
